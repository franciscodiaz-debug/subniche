# Current State

This documentation package reflects product-owner decisions through the latest rules discussion.

## Current MVP assumptions

- MVP is multi-niche capable.
- Initial launch likely focuses on roughly three seeded niches.
- Communities are not in MVP.
- Discussions, comments, reactions, and forums are not in MVP.
- Trade matching is required for MVP.
- Marketplace browse is niche-scoped and listing-focused.
- Search is niche-scoped and categorized by result type.
- Formal offers and counter-offers live inside messaging threads.
- Pending is manual owner-controlled availability, not automated offer lifecycle.
- Delete is destructive and replaces references with removed placeholders.
- Archive keeps private owner history but removes active/public surfaces.
- Super-admin control panel is included in MVP.
