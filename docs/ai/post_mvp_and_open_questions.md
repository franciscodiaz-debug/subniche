# Post-MVP and Open Questions

This file records features intentionally deferred from MVP.

## Communities — post-MVP

Communities are not in MVP.

Post-MVP community features may include:

- public communities,
- request-to-join communities,
- private/invite-only communities,
- community-only listings,
- community-specific listing visibility,
- community moderation/removal of listings,
- community owner/admin tools,
- community marketplace surfaces,
- community-specific trust signals,
- community discussions.

Earlier community-related ideas should be treated as post-MVP unless re-approved.

## Discussions/comments — post-MVP

Not in MVP:

- discussion posts,
- forums,
- item comments,
- collection comments,
- reactions,
- likes,
- public feeds.

Messaging is the MVP engagement layer.

## Transaction infrastructure — post-MVP

Not in MVP:

- payment processing,
- escrow,
- buyer/seller protection,
- shipping labels,
- tax calculation,
- checkout,
- insurance/protection products.

MVP negotiation and completion happen manually/off-platform.

## Reputation/trust expansions — post-MVP

Not in MVP:

- public reputation score,
- marketplace-history verification,
- social vouches,
- external badges,
- completed transaction history visible to public,
- moderator endorsements.

## Advanced trade features — post-MVP

Not in MVP:

- numeric match scores,
- Near Miss recommendation surfaces,
- fuzzy/AI-ranked match recommendations,
- automatic bundle matching,
- negative trade-interest filters,
- cross-niche formal offers,
- three-way trades,
- hard distance matching if not explicitly added later.

## Global owner dashboard — post-MVP

Future product may include a global control panel where users manage everything they own across niches.

MVP public-facing model remains niche-specific.

## Collection enhancements — post-MVP

Potential future features:

- archived collections,
- multiple collections per item,
- formerly owned public history,
- collection comments,
- more complex sharing/permission models.

For MVP, items can belong to only one collection and collections support delete only.

## Admin/CMS expansions — post-MVP

MVP includes a super-admin control panel, but avoid turning it into a full CMS unless needed.

Potential later tools:

- bulk taxonomy migration tools,
- richer audit logs,
- richer moderation workflows,
- community admin tools,
- advanced niche landing page builder.
